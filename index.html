Additional Information

Tools for web design

GitHub is used to provide website domain and record every text or syntax update. This is important for troubleshooting, as the developer can see what exact code changes has been committed for each update. If there is a problem with the new code, it is easy to go back to the old working version of the website through the commit history. Additional features such as repositories enable different teams to access different versions of the website and the pull request function also allows for easy collaboration with other users. The website was made using a template provided by bootstrap, as it saves time and effort from building from scratch and allows the designer to focus more on the content of the website rather than the HTML alone. Atom text editor was used with the aid of Google developer tool for editing the index HTML file and the css file. The Google developer tool pinpoints the corresponding code sections as user hovers over the website, which reduces time spent on low-level code searching. It also gives preview in terms of web and mobile platform, which caters for the accessibility of the website.

Website Design

(i) In my opinion, accessibility, visibility, device compatibility is important for a good website. It has to be aesthetically pleasing: allow room for white space, simple layout and easy to read fonts. It also has to be functional: fast loading speed, easy navigation, good quality content and section for contact.

(ii) Accessible allows any user to use any website on the internet. Despite if there is a condition that affects capabilities or what hardware and software is used. Adding alternative text in place of the image if the image fails to load, can allow screen readers to read the image to users who might miss it otherwise. It can also improve the SEO of the website. An accessible website should be keyboard friendly in terms of navigation, and users can go to all major features without having to use a mouse. If there is dynamic content on the website, such a slideshow, then it must be made aware to the assistive tools that there is a ‘live region’ so that users won’t miss the content. The contrast between text and background must be high enough to read easily and to ensure that everyone can distinguish between various elements on the page. To support users with visual impairment with resizing the text, avoid absolute units, such as specifying text size using pixels in web design. Instead, use relative sizes, which enable the text to scale depending on other content and screen size.

(iii) The General Data Protection Regulation protects users against data breaches as it keeps the customer data secure and make marketing communications clear. The footer of website must have privacy to provide transparency of how data is collected and used by the company and third parties. Customers have a ‘right to be forgotten’ so that they can have their details removed from a website and the database if they request it. Webmasters should therefore have a process in place that caters for this and also facilitate a way that users can request this, whether it mentioning it clearly in their privacy policy or elsewhere on the website. When users give personal data, they must show consent for the company to use it by opt-in boxes. These boxes should assume that users do not give consent to use information and require active ticking to give consent. To emphasize the security of customer data, website owners are required to keep all data secured in an encrypted environment. Adding https protocol helps to encrypt the data stored from the website.

(iv) To drive traffic to my website, I can incorporate search engine optimization (SEO), advertisement on social media platforms such as Instagram, Twitter, Facebook and Linkedin. I can also print QR code on product packaging to customers with after sale services.

(v) If additional time given, I can add functions to allow users to make account on the website to purchase product directly. This is important for expanding e-commerce.

Domain Setup

To set up a custom domain, firstly a domain name should be registered and configured with a Domain Name Systems(DNS) provider, such as Hurricane Electric Internet Services, NS1 etc. A DNS provider is a company that allows users to buy and register a unique domain name and connect that name to an IP (Internet Protocol) address by pointing your domain name to an IP address or a different domain name. Then the custom domain can be added to the GitHub Pages, in the repository settings.
